# xfinder 需求文档

## 一、产品概述

xfinder 是一个命令行本地文件搜索工具，灵感来自 Everything，但专注于终端使用场景。用户运行工具后，系统自动构建索引，随后可通过命令行输入查询条件，快速返回搜索结果。

---

## 二、核心特性

### 2.1 启动时构建索引

- 程序启动时自动扫描指定目录（默认为用户主目录或配置的目标路径）
- 构建完成后提示用户，进入搜索就绪状态
- 索引持久化到本地，下次启动时重新构建（无需增量更新）

### 2.2 三种索引模式

| 模式 | 说明 | 示例查询 |
|------|------|----------|
| **文件信息索引** | 索引文件名、路径、扩展名、大小、修改时间等元数据 | `resume.pdf`、`*.log`、`size>10MB` |
| **文件内容索引** | 对文本类文件进行全文索引，支持关键词检索 | `"TODO fixme"`、`import pandas` |
| **大模型索引** | 用 LLM 对文件生成自然语言描述，支持语义搜索 | `上个月写的关于用户登录的代码`、`有没有处理支付的文件` |

### 2.3 命令行交互模式

- 支持两种使用方式：
  - **一次性查询**：`xfinder "查询内容"` 直接返回结果后退出
  - **交互模式**：`xfinder` 进入 REPL，可连续输入多条查询
- 输出格式为结构化列表，包含文件路径、匹配摘要、相关度分数

## 三、功能需求

### 3.1 索引构建

- [ ] 支持配置扫描根目录（单个或多个）
- [ ] 支持配置排除目录（如 `.git`、`node_modules`、`__pycache__`）
- [ ] 支持配置排除文件类型（如 `.DS_Store`、`*.pyc`）
- [ ] 文件信息索引：必须支持
- [ ] 文件内容索引：可配置开关，默认对常见文本格式启用（`.txt`、`.md`、`.py`、`.js`、`.ts`、`.go`、`.java`、`.c`、`.cpp`、`.json`、`.yaml` 等）
- [ ] 大模型索引：可配置开关，默认关闭（需要用户提供 API Key）
- [ ] 索引存储在本地（如 `~/.xfinder/index/`），每次启动时重新构建

### 3.2 搜索功能

- [ ] **文件名搜索**：支持精确匹配、模糊匹配、通配符（`*`、`?`）
- [ ] **路径搜索**：支持路径片段匹配
- [ ] **扩展名过滤**：`type:pdf`、`type:py`
- [ ] **大小过滤**：`size>1MB`、`size<100KB`
- [ ] **时间过滤**：`modified:7d`（最近7天）、`modified:2024-01`
- [ ] **全文搜索**：在已索引的文件内容中检索关键词，支持短语搜索（加引号）
- [ ] **自然语言搜索**：用 LLM 理解查询意图，结合向量相似度返回语义相关文件
- [ ] **混合搜索**：自动判断查询类型，同时使用多种索引联合排序

### 3.3 输出格式

- [ ] 默认输出：文件路径 + 匹配摘要（高亮关键词）
- [ ] 支持 `--json` 参数输出 JSON 格式
- [ ] 支持 `--limit N` 控制返回条数（默认 20）
- [ ] 支持 `--sort [relevance|name|size|time]` 排序方式
- [ ] 全文/语义搜索结果显示匹配上下文片段（前后各 1 行）

### 3.4 配置文件

配置文件路径：`~/.xfinder/config.yaml`

```yaml
scan_paths:
  - ~/Documents
  - ~/Projects
exclude_dirs:
  - .git
  - node_modules
  - __pycache__
content_index:
  enabled: true
  extensions: [.txt, .md, .py, .js, .ts, .go, .java, .json, .yaml]
  max_file_size: 5MB
llm_index:
  enabled: false
  base_url: "https://api.openai.com/v1"  # OpenAI 兼容的 API 地址
  api_key: ""
  model: "gpt-4o-mini"
  embedding_model: "text-embedding-3-small"
```

## 四、非功能需求

| 项目 | 要求 |
|------|------|
| 索引构建速度 | 10 万文件以内，首次全量索引 < 60 秒 |
| 搜索响应时间 | 文件名/内容搜索 < 200ms；语义搜索 < 2s（含网络）|
| 存储占用 | 文件信息索引 < 50MB/10万文件；内容索引视文件体积而定 |
| 跨平台 | macOS、Linux 优先，Windows 可选 |
| 依赖 | 尽量减少外部依赖，可以打包为单一可执行文件 |

## 五、技术选型建议

| 组件 | 建议方案 | 备注 |
|------|----------|------|
| 开发语言 | Python | Python 生态丰富，开发效率高 |
| 文件信息索引 | SQLite | 轻量，查询灵活 |
| 全文索引 | SQLite FTS5 或 Whoosh (Python) | FTS5 无需额外依赖 |
| 向量索引（语义） | ChromaDB / FAISS / hnswlib | 本地运行，无需服务端 |
| LLM 嵌入 | OpenAI Embeddings / Ollama 本地模型 | 支持离线使用 |
| CLI 框架 | Click / Typer (Python) 或 cobra (Go) | |
| 配置解析 | PyYAML / viper (Go) | |

## 六、用户交互流程

```
$ xfinder

正在构建索引...
  扫描文件信息: 82,341 个文件  ✓
  构建全文索引: 12,847 个文件  ✓
  语义索引: 已禁用（配置 llm_index.enabled=true 开启）

索引就绪，输入查询内容（Ctrl+C 退出）：

> resume

结果（共 3 条，耗时 12ms）：
  1. ~/Documents/resume_2024.pdf          [文件名匹配]
  2. ~/Projects/hr-system/models/resume.py [文件名匹配]
  3. ~/Notes/career/resume_tips.md         [内容匹配: "...更新你的 resume 时需要注意..."]

> "def authenticate" type:py

结果（共 2 条，耗时 45ms）：
  1. ~/Projects/api/auth.py:23             [内容匹配: "def authenticate(user, password):"]
  2. ~/Projects/legacy/login.py:87         [内容匹配: "def authenticate(token):"]

> 上个月写的处理图片压缩的脚本

结果（共 1 条，耗时 1.2s）：
  1. ~/scripts/compress_images.py          [语义匹配: 相似度 0.87，修改于 2025-03-12]
```

## 七、里程碑规划

| 阶段 | 功能范围 | 目标 |
|------|----------|------|
| MVP | 文件信息索引 + 文件名搜索 + 基础过滤 | 可用的命令行搜索 |
| v0.2 | 全文索引 + 内容搜索 | 替代 `grep -r` |
| v0.3 | 大模型语义索引 + 自然语言查询 | 智能搜索体验 |
| v1.0 | 配置系统完善 + 打包分发 + 文档 | 可对外发布 |
